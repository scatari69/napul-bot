# 🛡️ Deadlock Team Bot

Бот для организации сборов игроков в Deadlock. Помогает собирать пати и вести честную статистику игроков.

## 🚀 Функционал

* **Управление командой:** Легкое добавление (`/addme`) и удаление (`/removeme`) себя из основного состава.
* **Умный сбор (`/tag`):** Бот тегает только тех, кто еще не ответил на текущий сбор. Имеет встроенную защиту от спама (Cooldown).
* **Автосбор (`/set autotag ЧЧ:ММ`):** Раз в сутки в заданное время бот сам тегает состав — без напоминаний руками.
* **Настройки для админов (`/set`):** Автосбор и правка состава (`/set team add/remove @ник`) прямо из чата.
* **Топ-статистика:** Автоматический учет сборов, топ-3 тегавших и топ-3 напулявших — все в `/stats`.
* **Автоматизация:** Ежедневная очистка списков сбора в 03:00 ночи, а если ждать до ночи не хочется — `/clear`.
* **Изоляция чатов:** У каждого чата свой состав, свой кулдаун, свой сбор и своя статистика — бота можно добавить в несколько групп.
* **Пасхалки:** Личный счетчик «напулов» заводится автоматически на каждого, кто зовет `/tag`, не будучи в составе. Привязка к `user_id`, работает только в выбранном чате.

## 🐳 Инструкция по установке (Docker)

Запуск бота в контейнере гарантирует сохранность статистики и стабильную работу 24/7.

### 1. Подготовка
Создайте папку проекта и поместите туда файлы бота (`bot.py`, `requirements.txt`).
Создайте файл `.env` и укажите в нем ваш токен:
```env
BOT_TOKEN=ваш_токен_от_botfather
EGG_CHAT_ID=-1179357258      # чат, где работают пасхалки; по умолчанию 1179357258
ADMIN_ID=174100936           # кто управляет пасхалками
TZ=Europe/Kyiv               # по нему бот сбрасывает списки сбора в 03:00
```

| Переменная | Обязательна | Описание |
|---|---|---|
| `BOT_TOKEN` | да | Токен от [@BotFather](https://t.me/BotFather). Без него бот не стартует |
| `EGG_CHAT_ID` | нет | Чат с пасхалками, см. раздел ниже |
| `ADMIN_ID` | нет | `user_id` тех, кому доступна команда `/clear`. Несколько — через запятую или пробел. Если не задать, право получают все админы чата |
| `TZ` | нет | Часовой пояс контейнера. Без него время внутри — UTC, и ночной сброс уедет |
| `DATA_DIR` | нет | Куда класть `state.json`. В Docker задан в compose (`/app/data`) |

### 2. Файлы настройки

`Dockerfile` и `docker-compose.yaml` уже лежат в репозитории. Код живет в `/app`, данные — в `/app/data`:

```YAML
volumes:
  - ./data:/app/data      # данные переживают пересборку
  - ./bot.py:/app/bot.py:ro   # код поверх образа — обновляется без --build
```

> ⚠️ Монтировать том прямо в `/app` нельзя — он перекроет весь код из образа. Поэтому подключаем подпапку `data` и отдельный файл `bot.py`.

Путь к данным задается переменной `DATA_DIR` (по умолчанию — текущая папка, в контейнере — `/app/data`), так что бота можно запускать и без Docker: `python bot.py`.

### 3. Запуск

Выполните команду в терминале:

```Bash
docker compose up -d --build
```

### 🔄 Обновление

`bot.py` примонтирован в контейнер, поэтому обновить код можно без пересборки:

```Bash
git pull
docker compose restart      # или кнопка Restart в Dockge
```

Пересборка (`docker compose up -d --build`) нужна только когда меняется `requirements.txt` или `Dockerfile`.

> В Dockge код собирается из папки стека (обычно `/opt/stacks/<имя>`). Делай `git pull` именно там, иначе контейнер подхватит старый `bot.py`. Проверить путь: `docker inspect deadlock_bot -f '{{index .Config.Labels "com.docker.compose.project.working_dir"}}'`.

### 💾 Данные

Всё состояние хранится в одном файле `state.json` внутри `DATA_DIR`, разбитое по `chat_id`. Запись атомарная (через временный файл + `os.replace`), правки идут под общей блокировкой — параллельные нажатия кнопок не затирают друг друга.

Старый формат (`stats.json`, `team.json`, `user_statistics.json`) переносится автоматически при первом запуске: так как `chat_id` в нем не сохранялся, данные достаются первому чату, который обратится к боту после обновления. Старые файлы не удаляются — можно оставить как бэкап.


### 🥚 Пасхалки

Пасхалка — это личный счетчик: когда человек **не входящий в состав** зовет `/tag`, бот вместо «403» отвечает «Имя напулял (уже N раз)».

Счетчик заводится сам при первом таком `/tag` — отдельный на каждого не-игрока, без всякой настройки. Имя берется из профиля (`first_name`, иначе `@username`, иначе `id<номер>`). Топ-3 напулявших показывается в `/stats`.

Привязка идет к числовому `user_id`, а не к `@username`, — ник можно сменить, id остается прежним. Поэтому пасхалка достается и тем, у кого username вообще не выставлен. Работают пасхалки только в одном чате, он задается переменной `EGG_CHAT_ID` (по умолчанию `1179357258`); в остальных чатах `/tag` от постороннего по-прежнему получает «403».

Id чата можно писать в любом виде — `1179357258`, `-1179357258` или `-1001179357258`: бот сравнивает без знака и без служебного префикса `-100`, которым Telegram помечает супергруппы.

Счетчики прежних захардкоженных пасхалок (Славик и Тексер) не теряются: как только кто-то с таким же именем (по профилю) впервые зовет `/tag`, накопленное число переезжает на его личную пасхалку. Пока этого не произошло, старые счетчики видны в `/stats`.

### ⏰ Автосбор и настройки (`/set`)

Админская команда `/set` меняет настройки чата. Доступ — как у `/clear`: при заданном `ADMIN_ID` только эти `user_id`, иначе любой админ чата.

```
/set autotag 20:00        — включить ежедневный автосбор на 20:00
/set autotag disable      — выключить автосбор
/set autotag              — показать текущее состояние
/set team add @ник        — добавить игрока в состав
/set team remove @ник     — убрать игрока из состава
```

Автосбор раз в сутки в указанное время делает то же, что `/tag`: тегает игроков, которые еще не отметили «+» в текущем сборе, и постит кнопки. Если состав пуст или все уже отметились — бот молчит. Время — по часовому поясу контейнера (`TZ`), тому же, по которому идет ночная очистка в 03:00. Настройка своя у каждого чата.

### 🛠 Команды бота


* /addme — Добавиться в список игроков.
* /removeme — Удалиться из списка.
* /team — Посмотреть текущий состав команды.
* /tag — Начать сбор (тегнуть всех, кто еще не отметил "+" в текущем сборе).
* /set — Автосбор и правка состава (админы): `autotag`, `team add/remove`.
* /clear — Очистить список плюсов и минусов, не дожидаясь ночи (админы).
* /stats — Общая статистика, топ-3 напулявших и топ-3 тегавших.
* /whoami — Показать свой user_id и chat_id (или чужой, если ответить на сообщение).
