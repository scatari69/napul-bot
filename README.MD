# 🛡️ Deadlock Team Bot

Бот для организации сборов игроков в Deadlock. Помогает собирать пати, вести честную статистику "напуляторов" и организовывать группы в М+.

## 🚀 Функционал

* **Управление командой:** Легкое добавление (`/addme`) и удаление (`/removeme`) себя из основного состава.
* **Умный сбор (`/tag`):** Бот тегает только тех, кто еще не ответил на текущий сбор. Имеет встроенную защиту от спама (Cooldown).
* **Топ-статистика:** Автоматический учет успешных сборов и личный рейтинг игроков, доступный через `/stats`.
* **М+ Сборы (`/mplus`):** Интерактивная система распределения ролей (Танк, Хил, ДД) с кнопками и автоматическим уведомлением о сборе группы.
* **Автоматизация:**
    * Ежедневная очистка списков в 03:00 ночи.
    * Таймаут сбора через 30 минут бездействия.
    * Автоматическая очистка заброшенных М+ сборов.
* **Изоляция чатов:** У каждого чата свой состав, свой кулдаун, свой сбор и своя статистика — бота можно добавить в несколько групп.
* **Пасхалки (`/egg`):** Настраиваемые счетчики «напулов» для отдельных людей, привязанные к `user_id` и живущие только в выбранном чате.

## 🐳 Инструкция по установке (Docker)

Запуск бота в контейнере гарантирует сохранность статистики и стабильную работу 24/7.

### 1. Подготовка
Создайте папку проекта и поместите туда файлы бота (`bot.py`, `requirements.txt`).
Создайте файл `.env` и укажите в нем ваш токен:
```env
BOT_TOKEN=ваш_токен_от_botfather
```

### 2. Файлы настройки

`Dockerfile` и `docker-compose.yaml` уже лежат в репозитории. Код живет в `/app`, а данные — в `/app/data`, поэтому том монтируется именно в подпапку:

```YAML
volumes:
  - ./data:/app/data
```

> ⚠️ Монтировать том прямо в `/app` нельзя — он перекроет `bot.py` из образа, и контейнер не запустится.

Путь к данным задается переменной `DATA_DIR` (по умолчанию — текущая папка, в контейнере — `/app/data`), так что бота можно запускать и без Docker: `python bot.py`.

### 3. Запуск

Выполните команду в терминале:

```Bash
docker compose up -d --build
```

### 💾 Данные

Всё состояние хранится в одном файле `state.json` внутри `DATA_DIR`, разбитое по `chat_id`. Запись атомарная (через временный файл + `os.replace`), правки идут под общей блокировкой — параллельные нажатия кнопок не затирают друг друга.

Старый формат (`stats.json`, `team.json`, `user_statistics.json`) переносится автоматически при первом запуске: так как `chat_id` в нем не сохранялся, данные достаются первому чату, который обратится к боту после обновления. Старые файлы не удаляются — можно оставить как бэкап.


### 🥚 Пасхалки

Пасхалка — это личный счетчик: когда человек **не входящий в состав** зовет `/tag`, бот вместо «403» отвечает «Имя напулял (уже N раз)».

Привязка идет к числовому `user_id`, а не к `@username`, — ник можно сменить, id остается прежним. Работают пасхалки только в одном чате, он задается переменной `EGG_CHAT_ID` (по умолчанию `1179357258`); в остальных чатах команда `/egg` отвечает, что пасхалки не включены, и подсказывает `chat_id` текущего чата.

Id чата можно писать в любом виде — `1179357258`, `-1179357258` или `-1001179357258`: бот сравнивает без знака и без служебного префикса `-100`, которым Telegram помечает супергруппы.

Управляют пасхалками админы чата:

```
/egg list                    — показать все пасхалки чата
/egg add Имя                 — в ответ (reply) на сообщение нужного человека
/egg add <user_id> Имя       — если человек сейчас не пишет
/egg del <user_id>           — убрать пасхалку
```

`user_id` можно подсмотреть командой `/whoami` — своя, либо в ответ на чужое сообщение.

Счетчики прежних захардкоженных пасхалок (Славик и Тексер) не теряются: добавь пасхалку с тем же именем — накопленное число переедет на нее автоматически. Пока этого не произошло, старые счетчики видны в `/stats` и в конце `/egg list`.

### 🛠 Команды бота


* /addme — Добавиться в список пуляторов.
* /removeme — Удалиться из списка.
* /team — Посмотреть текущий состав команды.
* /tag — Начать сбор (тегнуть всех, кто еще не отметил "+" в текущем сборе).
* /mplus — Создать интерактивный опрос для сбора группы в М+.
* /stats — Посмотреть общую статистику и топ лучших пуляторов.
* /whoami — Показать свой user_id и chat_id (или чужой, если ответить на сообщение).
* /egg — Управление пасхалками (только админы, только в EGG_CHAT_ID).
